Delegates, Lambdas, and Events

This section explains how delegates, lambda expressions, and events work in C#. They enable flexible code design, event-driven programming, and inline method definitions.

Delegates

A delegate is a type-safe function pointer that defines a method signature. For example:

public delegate void GreetDelegate(string name);

public class Greeter
{
    public void Greet(string name)
    {
        Console.WriteLine($"Hello, {name}!");
    }
}

You can assign a method to the delegate and call it:

GreetDelegate greetHandler = new Greeter().Greet;
greetHandler("Alice");

Lambda Expressions

Lambda expressions provide a concise way to write anonymous methods. They are especially useful with delegates. For example:

GreetDelegate greetHandler = (name) => Console.WriteLine($"Hello, {name}!");
greetHandler("Bob");

Events

Events use delegates to enable a class to notify other classes or objects when something occurs. For example:

public class Button
{
    public event Action Clicked;

    public void OnClick()
    {
        if (Clicked != null)
            Clicked();
    }
}

public class Program
{
    public static void Main()
    {
        Button btn = new Button();
        btn.Clicked += () => Console.WriteLine("Button was clicked!");
        btn.OnClick();
    }
}

Extension Methods

Extension methods allow you to add new methods to an existing type without modifying it. For example:

public static class StringExtensions
{
    public static bool IsNullOrEmpty(this string str)
    {
        return string.IsNullOrEmpty(str);
    }
}

// Usage:
string sample = "";
bool isEmpty = sample.IsNullOrEmpty();
